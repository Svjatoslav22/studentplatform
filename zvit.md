    ## 1. ВСТУП

    Передипломна практика була спрямована на розробку та впровадження веб-платформи для соціальної взаємодії студентів закладів професійної (професійно-технічної) освіти. Даний проєкт реалізований як частина навчального процесу на освітньому рівні магістра за спеціальністю в сфері інформаційних технологій.

    Під час практики сформовано комплексне рішення, яке об'єднує функції перегляду розкладу занять, обміну навчальними матеріалами, публікації запитань та відповідей, а також внутрішнього месенджера для оперативної комунікації між студентами.

    ### 1.1 Актуальність теми

    У сучасних умовах розвитку цифрових технологій та освіти виникла нагальна необхідність у створенні засобів ефективної комунікації студентів професійно-технічних закладів. Традиційні методи взаємодії (електронна пошта, SMS) не дозволяють студентам оперативно обмінюватися інформацією про розклад, завдання та навчальні матеріали.

    Розроблена платформа вирішує цю проблему, надаючи:
    - Централізований доступ до розкладу занять
    - Єдиний простір для обміну питаннями та знаннями
    - Зручний вбудований месенджер
    - Можливість доступу з будь-якого пристрою через веб-браузер

    ### 1.2 Мета та завдання практики

    **Мета:** Розробити та впровадити веб-платформу для соціальної взаємодії студентів з функціоналом управління розкладом, обміну матеріалами та комунікацією.

    **Завдання:**
    1. Проаналізувати вимоги до системи та її архітектури
    2. Спроєктувати структуру бази даних
    3. Розробити backend-сервіс на базі Node.js та Express
    4. Розробити frontend-інтерфейс на базі React
    5. Реалізувати функціонал управління розкладом та Т&О
    6. Розробити систему обміну запитаннями та відповідями
    7. Реалізувати вбудований месенджер для комунікації
    8. Забезпечити персоналізацію профілю користувача
    9. Локалізувати інтерфейс (українська та англійська мови)
    10. Протестувати та задокументувати рішення

    ## 2. АНАЛІЗ ЗАВДАННЯ

    ### 2.1 Вимоги до системи

    Система повинна задовольняти функціональні та нефункціональні вимоги:

    #### Функціональні вимоги:
    - Автентифікація та реєстрація користувачів
    - Перегляд розкладу занять по спеціальностям та курсам
    - Публікація запитань та відповідей з теговою системою
    - Месенджер з фільтрацією за спеціальностями та курсами
    - Збереження історії повідомлень
    - Управління профілем користувача
    - Доступ до навчальних матеріалів (книги, збірники)

    #### Нефункціональні вимоги:
    - Швидкодія: час завантаження сторінки менше 3 секунд
    - Масштабованість: можливість обслуговування 500+ одночасних користувачів
    - Безпека: защита від SQL-injection та XSS атак
    - Кроссбраузерність: підтримка сучасних браузерів (Chrome, Firefox, Safari)
    - Локалізація: двомовний інтерфейс (українська, англійська)
    - Доступність: адаптивний дизайн для мобільних пристроїв

    ### 2.2 Аналіз існуючих рішень

    На ринку існують рішення подібного типу:
    - **Slack** — корпоративний месенджер з широким функціоналом (дороге)
    - **Discord** — комунікаційна платформа (заснована на комунітях)
    - **Moodle** — LMS система (складна, надлишкова для поточних завдань)
    - **Google Classroom** — академічна платформа (вимагає Google аккаунту)

    Розроблювана платформа вирішує точно визначений набір задач освітньої установи без надлишкового функціоналу.

    ### 2.3 Технологічний стек

    **Frontend:**
    - React 17 — бібліотека для побудови користувацького інтерфейсу
    - Material-UI 4 — набір компонентів для дизайну
    - Lucide React — набір іконок з відкритим кодом
    - SCSS — препроцесор для стилізації
    - Axios — HTTP клієнт для комунікації з backend

    **Backend:**
    - Node.js — runtime для виконання JavaScript на сервері
    - Express.js — веб-фреймворк
    - MySQL 8 — реляційна база даних
    - mysql2/promise — драйвер для MySQL з підтримкою async/await

    **Інструменти розробки:**
    - Concurrently — запуск backend та frontend одночасно
    - Cross-env — встановлення змінних середовища
    - npm — менеджер пакетів

    [Вставити скрін 16: SQL CREATE TABLE statements]

    ```sql
    CREATE TABLE users (
    id INT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(255) UNIQUE NOT NULL,
    password VARCHAR(255) NOT NULL,
    email VARCHAR(255),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    CREATE TABLE timetable (
    id INT PRIMARY KEY AUTO_INCREMENT,
    department VARCHAR(100),
    section VARCHAR(10),
    semester INT,
    subject VARCHAR(255),
    day_of_week VARCHAR(20),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
    );

    CREATE TABLE questions (
    id INT PRIMARY KEY AUTO_INCREMENT,
    user_name VARCHAR(255),
    message TEXT,
    tags JSON,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP
    );

    CREATE TABLE question_answers (
    id INT PRIMARY KEY AUTO_INCREMENT,
    question_id INT,
    user_name VARCHAR(255),
    answer_text TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP,
    FOREIGN KEY (question_id) REFERENCES questions(id) ON DELETE CASCADE
    );
    ```
